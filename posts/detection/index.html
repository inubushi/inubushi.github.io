Image Detection Vs. Image Classification

May 31, 2020

In the previous post, we learned what Image Classification is. To remind in simple terms, Image classification is the task of identifying the category of the dominant object in an image (photo, or camera preview). There is a list of categories that the classifier knows how to distinguish between, and the dominant object HAS TO belong to one of them, for meaningful results.

Image classification does have a variety of applications. For example, It can be used on a production line, to see if the product under the camera is “good” or “defective”. But in some applications, classification is not good enough.

Suppose we have a surveillance camera that we want to connect to an AI, to automatically analyze the human behavior in the view of the camera. Image classification can tell you that a person is currently dominating the image, when one does. It may not be able to identify that one or more persons are present in the view, until one gets sufficiently close to the camera. Information about the number of people will not be available. Obviously, this does not fit many applications in automatic surveillance.

Image detection is the technology required for this kind of tasks. In image detection, the AI detects the positions of all instances of known objects. The image below shows an example of detecting different obects in a photo.

The following are some importnt observations coming from this image:

  • each detected object belongs to one of the classes that the detector has been trained to identify

  • multiple objects from the same class are detected

  • for each detected object, the bounds of a rectangle around the object (called bounding box) is provided as output

  • the bounding boxes can overlap

  • the area outside the bounding boxes is considered as background by the detector

A good example for image detection is available here under Apple’s documentation on the Vision framework. This demo runs on live camera preview and recognizes several categories of breakfast items.

However, the example is a bit difficult to understand for a beginner, because of more complicated code fro running on live preview. The number of things that it can detect is also limited.
